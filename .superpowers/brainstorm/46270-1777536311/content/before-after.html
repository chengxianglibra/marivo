Current vs Proposed Architecture

Three-object model → Single datasource object

Current: 3 Objects
Source (metadata authority)
source_type, authority.connection,
authority.synthetic_catalog,
sync.mode, intrinsic_capabilities,
policy (allow_live_browse, allow_sync)
↕ mapping
Mapping (projection contract)
source_id, engine_id, priority,
catalog_mappings[]:
  authority_catalog → execution_catalog,
  default_schema
↕ mapping
Engine (execution authority)
engine_type, connection,
auth (mode, username_source, fallback_username),
default_namespace,
intrinsic_capabilities,
deployment_capabilities,
policy (allowed_step_types, required_policy_support)
Proposed: 1 Object
Datasource (unified)
datasource_type (duckdb | trino),
display_name,
connection { host, port, catalog, ... },
sync.mode,
policy {
  allow_live_browse,
  allow_sync,
  allow_identity_reuse,
}
capabilities → derived from type (read-only)
Removed:
Mapping object, synthetic_catalog,
engine auth (mode/username_source/fallback_username),
default_namespace, deployment_capabilities,
intrinsic_capabilities (writable), catalog_mappings
