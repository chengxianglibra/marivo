Marivo 是一个 Python library —— 不是托管服务，也不是 chat UI —— 它让一个数据仓库变成
AI agent 能够*可靠*分析的对象。它为 agents 提供三样它们原本缺失的能力：固定每个 metric
和 dimension 含义的类型化 **semantic layer**、由类型化 operators 组成的**以 metric 为中心
的分析流程**，以及每一个结论背后**可审计的 evidence 链**。

Marivo 不是 text-to-SQL wrapper。Python declarations 是契约，ibis expressions 是执行
语言，typed frames 是分析步骤之间的边界 —— 这样 agent 是在组合可信的构件，而不是生成原始
SQL 然后寄希望于它正确。

## 为什么 agentic data analysis 需要 Marivo

把 agent 直接指向一个原始仓库、让它写 SQL，失败模式是可预见的：它会臆造 join、悄悄漏掉一个
必需的过滤条件、误读含义不清的列，并给出一个谁也无法核验的自信答案。每次查询都从零开始，于是
什么都不会累积，什么都无法审核。

Marivo 用一份 agent 必须遵循的契约，取代“生成 SQL 然后碰运气”：

| 没有 semantic runtime | 有了 Marivo |
| --- | --- |
| 每次查询都靠猜表名和列名 | 可信的 **semantic refs**（`sales.revenue`），携带人工编写的含义与 guardrails |
| 每次都重新推导定义 | 一份纳入版本控制的**声明式契约**，在 agents 与 sessions 之间共享 |
| 自由拼写的 SQL，错误以错误数字的形式暴露 | **类型化 intents 与 frames** —— 非法步骤在任何 backend 操作之前就大声报错 |
| 无法核验的结论 | 把每个结果链接回其输入的 **evidence 链** |
| 半成品模型照样被分析 | 阻止未完成 semantics 进入分析的 **readiness gate** |

最终得到的，是一个基于*含义*而非列名来推理的 agent，并且整个推理过程处在一个人类可以审核、
批准与重放的回路中。

## 它如何工作

1. **声明**：在 `models/datasources/` 中声明 datasources，在 `models/semantic/` 中声明
   semantics（domains、entities、dimensions、metrics）—— 使用 `marivo.datasource`
   （`md`）和 `marivo.semantic`（`ms`）。
2. **加载并设闸**：`ms.load()` 构建 catalog；`ms.readiness()` 在 agent 所需的每个对象
   都完整且可达之前阻止分析。
3. **打开 session**：从一个 guiding question 出发，并从 catalog 解析出一个可信 metric。
4. **串联 typed intents** —— `observe → compare → decompose` 等 —— 每一步返回一个 typed
   frame 并留下 evidence 痕迹。

## 第一次分析

```python
import marivo.analysis as mv

session = mv.session.get_or_create(name="revenue-check", question="Why did Q4 drop?")
catalog = session.catalog
revenue = catalog.get("sales.revenue")
region = catalog.get("sales.orders.region")

current = session.observe(revenue, timescope={"start": "2026-10-01", "end": "2027-01-01"}, grain="month", dimensions=[region])
baseline = session.observe(revenue, timescope={"start": "2025-10-01", "end": "2026-01-01"}, grain="month", dimensions=[region])

delta = session.compare(current, baseline)
attribution = session.decompose(delta, axis=region)
attribution.show()
```
