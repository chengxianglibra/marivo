Marivo 0.2.8 是 v0.2 系列的 patch release，聚焦 agent-facing analysis workflows。
它新增只读 `marivo doctor` 诊断命令，将 `observe(...)` 扩展到同一 scope 下的
multi-metric observations，在 session knowledge 中记录有界 observation digests，并收紧
若干 observe planner 边界情况。

## 重点

- `marivo doctor` 现在可以在 agent 开始编写或运行 analysis work 前，执行只读的
  environment、project、semantic 和 datasource checks。它可以检查 semantic files，并可选验证
  datasource connectivity。
- `observe(...)` 现在接受同一 scope 下的非空 metric 序列，用于 multi-metric
  observations。Marivo 会融合兼容的执行工作，投影请求的 metric columns，并将本次 observation
  记录为一个 evidence step。
- Session knowledge 现在包含 `observe(...)` 和 `derive_metric_frame(...)` commits 的有界
  observation digests。Agents 可以复查已经观察到的内容，而不必重新读取完整 frames。

## 变更

- `marivo init` 现在会把 bundled skills 安装到共享 `.agents/skills` 目录，同时保留 Claude
  Code 和 Codex compatibility targets。
- Analysis intents 现在会通过 frames、lineage 和 help output 携带显式 purpose，使 evidence
  能说明一个结果为何被产出。

## 修复

- Fanout-side `where` slices 现在会在 merge-grain distinct planning 前应用，从而让
  cross-dataset observe 结果与 authored filters 保持一致。
- Base observe queries 会在执行前裁剪未使用的 projections，减少不必要的 selected columns。
- Slice collection shorthand 会在执行前被一致地 normalize。
- ClickHouse datasource authoring 会在不支持 autogenerated SQLAlchemy sessions 的场景中禁用该行为。

## 文档

- 新增 multi-metric observations、observation digests、`marivo doctor` 和共享 `.agents/skills`
  install target 的 docs 与 examples。
- 新增 multi-metric observe 和 cumulative metrics 的 design specs。

## 安装

```bash
pip install marivo
```

按 datasource 选择对应 backend extra：

```bash
pip install "marivo[duckdb]"
pip install "marivo[mysql]"
pip install "marivo[trino]"
pip install "marivo[postgres]"
pip install "marivo[clickhouse]"
pip install "marivo[all]"
```

Marivo 0.2.8 需要 Python 3.12 或更高版本。
