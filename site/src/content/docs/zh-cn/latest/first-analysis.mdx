## 开始分析

使用项目中已经确认业务含义、并且当前 catalog 可以加载的指标。如果指标缺失或口径仍有
疑问，先让智能体使用 `marivo-semantic` 解决明确的缺口。

你不需要编写 Python、选择算子或安排检查步骤。只要说明问题、指标、比较范围，以及希望
优先检查的因素：

> 使用 `marivo-analysis`，分析 2026 年第四季度收入相比 2025 年同期下降的原因。
> 使用已经确认的 `sales.revenue` 指标，按月比较，并优先检查地区差异。
> 最后给出结论、关键证据和限制。

## Marivo 会处理什么

智能体会读取相关语义定义，解析并验证 live semantic inputs，在同一个 analysis session 中
执行合适的类型化分析，并保留支持结论的结果和证据。遇到语义缺口时，它应返回
`marivo-semantic`，而不是在分析过程中悄悄修改口径；新建或修改的 objects 会在 authoring
closeout 中完成 readiness。

智能体通过 `python -m marivo help analysis`（使用项目解释器）和 focused
`mv.help(...)` 主题发现实时分析契约。结构化错误携带类型化修复指引，类型化边界
（`session.observe` 入、`frame.to_pandas()` / `md.raw_sql` 出）确保自定义分析不会断开证据链。

这些执行细节由 live help、frames 和结构化错误负责，不需要写进你的提示词。

## 继续之前需要检查什么

接受结论或要求进一步调查前，请确认：

- 指标含义与你关心的业务结果一致；
- 时间范围和分析群体（包括排除范围）正确；
- 比较方式和优先检查的因素具有业务意义；
- 每项重要判断都有 Marivo 结果或证据支持；
- 限制、质量问题和未解决的问题都已明确说明；
- 智能体没有悄悄修改语义层。
